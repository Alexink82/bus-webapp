# Чек-лист ручной проверки (Bus Booking Webapp)

Проверка сценариев пассажира, диспетчера и админа для обеспечения стабильности и работоспособности. Отмечайте выполнение: [ ] не проверено, [x] пройдено, [!] провалено.

---

## Подготовка

- [ ] Сервер запущен (`cd backend && uvicorn main:app --reload --host 0.0.0.0`)
- [ ] БД доступна, выполнено `seed_db.py` (маршруты, FAQ при необходимости)
- [ ] В .env заданы: `DATABASE_URL`, при проверке Telegram — `BOT_TOKEN`, `ADMIN_IDS`; для диспетчера — запись в таблице `dispatchers` или bot_roles; для кнопки в уведомлениях — `WEBAPP_URL` (HTTPS в проде)
- [ ] Открыть приложение: в браузере `http://localhost:8000` или через Telegram Web App (для проверки initData)
