## Что уже сделано (закоммичено)

Последний релевантный коммит по UI: **`bda84a6`** — «UI: адаптивный dashboard, sidebar навигация, диспетчер/админка» (ветка `main`, запушено на `origin`).

### Пассажирское приложение
- Оболочка **`app-shell`** + класс **`has-app-shell`** на страницах с табами: `webapp/index.html`, `profile.html`, `faq.html`, `success.html`.
- **≥1025px**: навигация слева (бывший нижний tab-bar), контент справа; **max-width 1200px**, центрирование.
- Главная: **двухколоночный grid** — форма/результаты слева, блок **«Актуальная информация»** справа (sticky).
- Форма поиска: **`search-form--booking`**, группа **`search-form__input-group`**, ряд маршрута **`search-form__route-row`**, действия в **`search-form__actions`**.
- Глобально **`--radius: 8px`**, тени **`--shadow-card-soft`** / **`--shadow-surface`**.

### Диспетчер
- Три колонки на **≥1024px**: **250px | flex | 300px**, боковые **sticky**; на узком экране боковые скрыты, кнопки «Фильтры» / «Моя активная заявка».
- Сессия и выход в **`<details class="dispatcher-meta-panel">`**.
- Верхняя панель **`dispatcher-topbar--dashboard`**: статистика (`statTotal`, `statNew`, `statOverdue`), поиск, SLA.
- Карточки заявок переразмечены (маршрут/время/цена/действия), секция просроченных с **пастельным фоном**.

### Админка
- Сессия в **`<details class="admin-meta-panel">`**.
- Сетка **max-width 1200px**, сайдбар **250px**, скругления **8px** у навигации.

### Сборка
- В репозитории обновлён **`dist/`** (после `npm run build`).

## Что НЕ довели до полного совпадения с изначальным ТЗ

1. **Breakpoint «мобильный таб только до 768px»**  
   Сейчас переключение **sidebar ↔ нижний tab-bar** на **1025px**, а не 768px. Если нужно строго: до 768 — снизу, 769–1024 — уточнить (снизу или слева).

2. **Мобильные фильтры диспетчера — «Drawer»**  
   Реализовано через **модалку** (`showAppModal` в `dispatcher.js`), не отдельная выезжающая панель.

3. **Правый виджет на главной: «Погода, Новости, FAQ»**  
   В сайдбаре по сути погода + канал; отдельного блока FAQ в колонке **не добавляли** (FAQ остаётся отдельной вкладкой).

4. **«Все отступы кратны 8px» по всему приложению**  
   Сделаны ключевые блоки и переменные, **полного аудита** всех экранов не было.

5. **Админка**  
   Не повторяли полностью «как диспетчер» (отдельная строка статистики в шапке и т.д.) — только компактная сессия и layout/density.
