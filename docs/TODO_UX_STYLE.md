# Задачи на будущее: стиль и UX

Зафиксировано по результатам тестирования. К выполнению — когда будет удобно.

## 1. Единый стиль приложения (админка и остальное)

**Проблема:** Админ-панель визуально отличается от остального приложения (другая цветовая схема, типографика, кнопки).

**Где:** `admin.html` подключает `css/design-system.css`; основное приложение использует `main.css` и `telegram-theme.css`.

**Задача:** Привести оформление админки к общему стилю приложения (те же переменные `--bg`, `--surface`, `--accent`, кнопки `.btn`, `.btn-primary` и т.д.) — либо перевести админку на `main.css` и доопределить только нужные классы, либо вынести общие токены в один файл и использовать их и в main, и в design-system.

---

## 2. Неделя в календаре с понедельника

**Проблема:** В календаре выбора даты (главная страница) неделя начинается с воскресенья.

**Где:** `webapp/index.html`, функция рендера календаря: массив `weekdays = ['Вс', 'Пн', 'Вт', ...]` и расчёт `startDay = first.getDay()` (0 = воскресенье).

**Задача:** Сделать начало недели понедельником: порядок подписей дней `['Пн', 'Вт', 'Ср', 'Чт', 'Пт', 'Сб', 'Вс']` и смещение для первой строки сетки: `startDay = (first.getDay() + 6) % 7` (понедельник = 0). Проверить все календари в приложении (главная, выбор даты в модалках и т.д.) и при необходимости применить ту же логику.

## 3. Единый образец кнопок (кнопка «Редактировать» в сохранённых пассажирах)

**Проблема:** Кнопка «Редактировать» в блоке сохранённых пассажиров (профиль) выделяется на фоне остальных кнопок приложения.

**Где:** `webapp/js/profile.js` — карточка пассажира: кнопка с классом `btn-link` («Редактировать») рядом с `btn btn-small btn-outline` («Удалить»). Стили ссылки-кнопки: `webapp/css/main.css` — `.btn-link`.

**Задача:** Привести кнопку «Редактировать» к общему образцу: либо оформить как `btn btn-small btn-outline` (как «Удалить»), либо ввести единый стиль вторичных текстовых действий (например, общий класс для «Редактировать» / «Подробнее») и использовать его по всему приложению.

## 4. Варианты на будущее (по правкам 2026-03-16)

### 4.1 Солнышко переключения темы

**Сейчас:** Кнопка темы остаётся слева в шапке; заголовок «Бронирование билетов» выровнен по центру экрана за счёт сетки (40px | 1fr | 40px).

**Варианты, если захочешь перенести:**
- **Оставить как есть** — привычно, под рукой.
- **Перенести в Настройки профиля** — в блок «Настройки» добавить пункт «Тема: Светлая / Тёмная / Системная»; из шапки убрать, тогда в шапке только заголовок и «Закрыть».
- **Вынести в меню ⋮** — в шапке одна кнопка «⋮» (или иконка), по нажатию выпадает меню: «Светлая тема», «Тёмная тема», «Закрыть». Освобождает место и центрирует заголовок без сетки.

### 4.2 Кнопки Активные / Предстоящие / Завершённые / Отменённые (профиль)

**Сделано:** Компактный вариант — уменьшены отступы и шрифт (`.profile-bookings-tabs`), все четыре кнопки помещаются в одну строку; кнопка «Обновить» перенесена под эти кнопки по центру.

**Если на очень узких экранах всё равно не влезет, варианты:**
- **Карусель** — горизонтальный скролл с лёгким «подсказом» (видна часть следующей кнопки), свайп влево/вправо. Плюс: все подписи полные. Минус: не каждый догадается пролистывать.
- **Сокращённые подписи** — «Актив.», «Предст.», «Заверш.», «Отмен.» (или иконки + короткий текст). Плюс: всё в одну строку без скролла. Минус: менее понятно с первого взгляда.
- **Выпадающий список** — одна кнопка «Фильтр: Активные ▼», по нажатию — список из четырёх пунктов. Плюс: экономия места. Минус: лишний тап и скрытие состояния.

Текущий компактный вариант оставлен как основной; при необходимости можно перейти на один из вариантов выше.
