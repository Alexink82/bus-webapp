# UI Style System Audit (единый визуальный стиль)

Дата: 2026-03-12

## Цель

Привести весь webapp к единому визуальному языку и проверить соответствие практикам:
- Nielsen Norman Group (UX-эвристики),
- Material Design,
- Apple Human Interface Guidelines,
- Microsoft Fluent Design.

## Что выровнено сейчас

1. **Единый theme-layer для всех страниц**
   - `telegram-theme.css` подключён также в:
     - `webapp/admin.html`,
     - `webapp/dispatcher.html`,
     - `webapp/booking.html`.
   Это убирает расхождение цветов/контрастов между экранами в Telegram WebApp.

2. **Единое применение сохранённой темы**
   - Для `admin.html` добавлен init-скрипт чтения `localStorage.theme` (как на остальных экранах).

## Проверка против крупных дизайн-систем

### 1) Nielsen Norman Group
- Видимость статуса системы: есть loader, maintenance overlay, явные состояния ошибок/пустых списков.
- Соответствие реальному миру: понятные названия маршрутов и статусов.
- User control & freedom: есть возврат/закрытие/отмена.
- Consistency & standards: усилено через общий theme-layer.

### 2) Material Design
- Цвет и типографика как системные токены (`main.css`/`design-system.css`).
- Иерархия карточек/панелей/кнопок сохранена.
- Состояния действий и ошибки читаемы.

### 3) Apple HIG
- Поддержан focus на читаемость и touch-friendly controls.
- Safe-area и mobile-first поведение уже используется.
- Тема и контраст в dark/light сценариях унифицированы.

### 4) Fluent Design
- Консистентность компонентов между экранами ролей.
- Предсказуемость навигации и action-элементов.
- Повышена визуальная целостность за счёт единого theme слоя.

## Рекомендованный финальный шаг (дальше)

Чтобы окончательно закрепить «один стиль на весь проект», следующий шаг —
вынести единый набор design tokens в один источник (например `design-system.css`),
а `main.css` и ролевые css держать как thin-layer (layout + page-specific).

## Чек-лист ручной визуальной приёмки

- [ ] Одинаковый цвет фона/текста/бордеров на `index`, `booking`, `profile`, `success`, `faq`, `dispatcher`, `admin`.
- [ ] Одинаковые состояния кнопок (`hover/active/disabled`) на всех экранах.
- [ ] Одинаковые заголовки и отступы в header-блоках.
- [ ] В Telegram и обычном браузере тема переключается одинаково.
- [ ] Нет «выпадающих» страниц со старой палитрой.
