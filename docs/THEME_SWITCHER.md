# Переключатель тем: реализация и рекомендации

## Что сделано

- **Две темы:** светлая и тёмная с разными палитрами и читаемостью.
- **Переключатель** в шапке на всех экранах (иконка ☀️/🌙), сохранение выбора в `localStorage`.
- **Логотип:** в светлой теме к нему применён `filter: invert(1)`, чтобы белый текст на чёрном фоне оставался читаемым на светлом фоне.
- **Ранняя инициализация:** небольшой inline-скрипт в `<head>` выставляет `data-theme` до отрисовки, чтобы не было мигания (FOUC).
- **Системная тема:** при первом заходе (если в `localStorage` ещё нет выбора) используется `prefers-color-scheme`.

## Рекомендации по доработкам

1. **Опция «Как в системе»**  
   Добавить третье состояние переключателя (например, «Система»), не сохранять в `localStorage` и при каждой загрузке брать тему из `prefers-color-scheme`. Так пользователь может всегда следовать системной теме.

2. **Отдельная страница настроек**  
   Вынести тему в «Настройки» или блок в профиле (радио: Светлая / Тёмная / Как в системе), а в шапке оставить только иконку для быстрого переключения между светлой и тёмной без «системы».

3. **Плавный переход**  
   Добавить `transition: background-color 0.2s, color 0.2s` для `body` и основных контейнеров, чтобы смена темы не была резкой.

4. **Цвет акцента в светлой теме**  
   Сейчас акцент чуть приглушён (#c73e54). При необходимости можно усилить контраст кнопок (например, ближе к #b03045) для лучшей заметности.

5. **Админ-панель**  
   Сейчас админка на design-system.css. При желании единообразия можно подключать `theme.js` и задавать `data-theme` на `html`, а в design-system описать переменные для `[data-theme="light"]` и тёмной темы.

6. **Тестирование контраста**  
   Проверить контраст в инструментах (например, WebAIM Contrast Checker) для всех пар «текст — фон» в обеих темах и при необходимости подправить `--text`, `--text-muted`, `--accent`.
