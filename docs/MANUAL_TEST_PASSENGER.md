# Ручная проверка: сценарии пассажира

Используйте этот чеклист после изменений в бронировании и профиле.

## Бронирование (booking.html)

- [ ] **Количество пассажиров**: − и + меняют число (1–10). Других кнопок в строке с числом нет.
- [ ] **«Добавить из сохранённых»**: кнопка видна только если в профиле есть сохранённые пассажиры. По нажатию открывается модалка выбора пассажира по слотам; «Подставить» заполняет выбранные слоты.
- [ ] **Телефон (шаг 2)**: можно полностью очистить поле (Backspace до пустого). При вводе цифр нет лишнего сдвига/замены; разделители не добавляются в конец, если цифр не хватает.
- [ ] **Шаг 1 → Шаг 2**: валидация ФИО и даты; при ошибках показываются сообщения.
- [ ] **Отправка заявки**: после успешного POST редирект на success.html с booking_id.

## Профиль (profile.html)

- [ ] **Добавить пассажира**: кнопка «Добавить» открывает модалку. Заполнение ФИО, даты рождения, паспорта; **Сохранить** — сохраняет (дата подставляется в скрытое поле из видимого, если IMask не сработал).
- [ ] **Редактировать пассажира**: «Редактировать» открывает модалку с данными; изменение и **Сохранить** — обновление, список перезагружается.
- [ ] **Модалка**: при открытой модалке добавления/редактирования фон страницы не прокручивается (ни снизу, ни сверху).
- [ ] **Перенести дату** (в «Подробнее» по заявке): при запрете (менее 2 ч для международного / 15 мин для междугороднего) — сообщение «только через диспетчера» и предложение связаться с поддержкой. При разрешённом переносе открывается модалка с календарём (выбор новой даты); после нажатия «Отправить заявку диспетчеру» заявка уходит на одобрение, показывается сообщение «Заявка на перенос отправлена диспетчеру. Ожидайте подтверждения.» и список заявок обновляется.

## Быстрая проверка телефона

1. Ввести 7–9 цифр (например 291234567).
2. Стереть все Backspace’ом — поле должно стать пустым.
3. Ввести снова и изменить середину номера — цифры не должны «прыгать» и подменяться.
