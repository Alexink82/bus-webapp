# План изменений UI по результатам анализа экранов

По скриншотам и коду выявлены проблемы. Ниже — приоритизированный план с конкретными правками в файлах.

---

## 1. Задвоение кнопки закрытия

**Проблема:** На части экранов видна и наша кнопка «✕», и штатная панель Telegram (с закрытием/меню) — создаётся впечатление двух кнопок закрытия.

**Причина:** В WebApp Telegram сам рисует верхнюю панель. Мы дополнительно показываем `.tg-close-btn` на index.html и success.html. В браузере без Telegram кнопки нет — её скрываем.

**Решение:**
- **В Telegram:** не показывать нашу кнопку «✕» — оставить только нативную панель Telegram.
- **Вне Telegram (браузер):** показывать одну нашу кнопку «Закрыть» в шапке.

**Файлы:**
- `webapp/index.html`, `webapp/success.html`: при инициализации проверять `window.Telegram && Telegram.WebApp && Telegram.WebApp.initData` — если есть (открыто в Telegram), скрывать `#tgCloseBtn` (`display: none`), иначе показывать.
- Единообразно: на всех страницах с шапкой (profile, booking, faq, admin, dispatcher) использовать один подход: одна кнопка закрытия справа, показывать только вне Telegram.

**Дополнительно:** Разнести кнопку и заголовок/логотип по разным зонам (см. п. 2), чтобы ни одна кнопка не перекрывала логотип.

## 2. Логотип не виден / кнопки перекрывают логотип

**Проблема:** В шапке нет отдельного логотипа; кнопка закрытия визуально пересекается с заголовком.

**Решение:**
- Ввести блок логотипа в шапке (слева или по центру). Варианты:
  - **A:** Картинка — положить `webapp/img/logo.svg` (или logo.png), в шапке `<img class="header__logo" src="img/logo.svg" alt="">`.
  - **B:** Пока без картинки — оставить эмодзи/текст в блоке `.header__brand` (логотип + название), структурно отдельно от кнопки закрытия.
- Сетка шапки: **слева** — логотип (или бренд), **по центру** — заголовок страницы, **справа** — одна кнопка «Закрыть» с фиксированным отступом (например `padding-right: 12px`), чтобы не заходить на зону логотипа/заголовка.
- Стили в `webapp/css/main.css`:
  - `.header` — flex, `align-items: center`, `justify-content: space-between`, минимальная высота, отступы.
  - `.header__logo` или `.header__brand` — размер и отступы так, чтобы на мобильном логотип был хорошо виден.
  - `.tg-close-btn` — `position: absolute; right: 12px; top: 50%; transform: translateY(-50%);` или через flex, чтобы не перекрывать центральный блок.

**Файлы:** `webapp/index.html`, `webapp/profile.html`, `webapp/faq.html`, `webapp/success.html`, при необходимости `webapp/booking.html` — единая разметка шапки; `webapp/css/main.css` — стили шапки и логотипа.

## 3. Меню слишком низко

**Проблема:** Нижняя навигация прилипает к самому низу экрана; на мобильных с вырезом/индикатором дома — неудобно.

**Решение:**
- Учесть safe area: у `.tab-bar` задать `padding-bottom: max(10px, env(safe-area-inset-bottom));` и при необходимости `bottom` не 0, а с отступом.
- Немного приподнять планку над «полом» экрана: например `bottom: 8px` или `margin-bottom: 8px` (если не fixed), чтобы визуально меню не было «слишком низко».
- Убедиться, что у `body` нижний отступ под контент не меньше высоты tab-bar + safe-area (уже есть `padding-bottom: calc(70px + env(safe-area-inset-bottom))` в telegram-theme.css — проверить, что 70px достаточно при новой высоте tab-bar).

**Файлы:** `webapp/css/main.css`, `webapp/css/telegram-theme.css` — класс `.tab-bar` и отступы body.

## 4. Вкладки «Администратор» и «Диспетчер» в нижнем меню

**Проблема:** У администратора и диспетчера в нижней навигации должны появляться соответствующие вкладки; сейчас только Бронь, Профиль, FAQ.

**Решение:**
- **Бэкенд:** добавить endpoint `GET /api/user/roles`, возвращающий `{ "is_admin": bool, "is_dispatcher": bool }`. Для авторизованного пользователя — по текущей логике (is_admin, get_dispatcher_route_ids). Без авторизации — 200 с `false, false`.
- **Фронт:** один раз при загрузке страниц с нижним меню (index, profile, faq) вызывать `GET /api/user/roles` (с initData/заголовками, как остальные запросы). По ответу динамически добавлять в `.tab-bar`:
  - при `is_dispatcher === true` — пункт «Диспетчер» со ссылкой на `dispatcher.html`;
  - при `is_admin === true` — пункт «Админ» со ссылкой на `admin.html`.
- Порядок пунктов, например: Бронь | Профиль | Диспетчер (если есть) | Админ (если есть) | FAQ.
- Общий скрипт для инициализации навигации (например `webapp/js/nav.js`), подключаемый на index, profile, faq. На страницах admin и dispatcher нижнее меню может быть другим или отсутствовать — по желанию оставить только «Назад в приложение» или дублировать те же вкладки.

**Файлы:**
- Backend: `backend/api/user.py` — новый endpoint `get_roles`.
- Frontend: `webapp/js/nav.js` (новый) — запрос ролей, вставка пунктов в `.tab-bar`; подключить в `index.html`, `profile.html`, `faq.html`.

## 5. Единый вид экранов (соответствие проекту)

**Проблема:** FAQ (и отчасти админка) выглядят иначе — другая цветовая схема и компоненты (design-system.css, светлая тема), остальные экраны — тёмная тема (main.css + telegram-theme.css).

**Решение:**
- Привести все публичные экраны к одной визуальной системе: **тёмная тема** и общие компоненты (main.css + telegram-theme.css).
- **FAQ:** подключать `main.css` и `telegram-theme.css` вместо только `design-system.css`. Разметку FAQ привести к той же структуре, что index/profile: общий `.container`, `.header` с тем же форматом (логотип/бренд, заголовок, кнопка закрытия), `.main`, единый `.tab-bar` с классами `.tab`, `.tab.active`. Стили блоков FAQ (поиск, список вопросов, блок «Написать в поддержку») переписать/дописать в main.css под тёмную тему или оставить минимальные переопределения.
- **Админ-панель:** можно оставить на design-system (отдельный «служебный» интерфейс) или перевести на main.css для полного единства — по приоритету после FAQ.
- **Диспетчер:** уже использует main.css + dispatcher.css — проверить, что шапка и при необходимости нижнее меню в одном стиле с остальными.

**Файлы:** `webapp/faq.html` (разметка и набор CSS), `webapp/css/main.css` (новые классы для FAQ-блоков в тёмной теме), при необходимости `webapp/admin.html` и стили.

## 6. Мобильная вёрстка (два последних скриншота — с телефона)

**Проблема:** На мобильном особенно заметны: задвоение кнопки закрытия, отсутствие логотипа, меню «слишком низко».

**Решение:**
- Выполнение пунктов 1–3 и 5 автоматически улучшит мобильный вид (одна кнопка, видимый логотип, учёт safe-area и отступов).
- Дополнительно проверить на мобильных:
  - размер кликабельных областей (кнопка закрытия, пункты tab-bar) не менее ~44px по высоте;
  - заголовок и логотип не обрезаются и не перекрываются на узком экране;
  - нижнее меню не перекрывается системной панелью (safe-area).
