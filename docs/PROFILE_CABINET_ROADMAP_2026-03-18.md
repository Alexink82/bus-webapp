# Roadmap: Personal Cabinet / Profile

Дата: 2026-03-18

## Цель
Превратить текущую вкладку `Профиль` из списка данных в личный кабинет поездки, где пользователь сразу видит:
- ближайшую поездку,
- что нужно сделать сейчас,
- как быстро повторить бронирование,
- где управлять пассажирами и контактами.

## Текущее состояние
- Есть вкладки `Мои брони / Пассажиры / Настройки`.
- Есть история заявок по статусам.
- Есть сохранённые пассажиры.
- Есть базовые настройки и телефон профиля.

## Главные UX-проблемы
- Нет верхнего блока `Главное` с ближайшей поездкой.
- Пользователю приходится искать нужную поездку в списках.
- Не хватает сценариев `повторить маршрут`, `открыть билет`, `что делать дальше`.
- Личный кабинет пока слабо помогает перед поездкой.

## Целевая структура
### Этап 1
- Добавить верхний блок `Главное` внутри профиля.
- Показать ближайшую поездку.
- Добавить быстрые действия:
  - `Подробнее`
  - `Открыть билет`
  - `Повторить маршрут`
  - `Поддержка`
- Добавить блок `Что сейчас важно`.
- Сохранить текущие вкладки и не ломать рабочий сценарий.

### Этап 2
- Добавить `историю статусов` в карточку поездки.
- Добавить `основного пассажира` и быстрый выбор пассажира для новой брони.
- Переработать `Настройки` в более пользовательскую структуру:
  - Контакты
  - Язык
  - Производительность интерфейса
  - Поддержка и правила

### Этап 3
- Добавить избранные маршруты.
- Добавить персональные подсказки и напоминания.
- Добавить travel-dashboard уровня commercial-ready:
  - ближайшая поездка,
  - быстрый повтор,
  - поддержка,
  - история,
  - готовность к поездке.

### Этап 4
- Добавить `центр уведомлений` в профиле.
- Добавить более понятный `журнал поездок`:
  - последняя активная,
  - последняя завершённая,
  - последняя отменённая,
  - быстрый переход к нужному списку.
- Добавить отдельный блок `готовность к рейсу`:
  - телефон,
  - пассажиры,
  - основной пассажир,
  - билет,
  - международный сценарий.

## Что внедрено
### Этап 1
- новый верхний обзорный блок в `Профиле`,
- ближайшая поездка,
- быстрые действия,
- полезная поддерживающая информация,
- без изменений backend API.

### Этап 2
- компактная история прогресса статуса в карточках поездок и в деталях заявки,
- основной пассажир с быстрым сценарием `Новая бронь`,
- автоматический приоритет основного пассажира при подстановке в `booking`,
- переработанные `Настройки` по блокам:
  - Контакты
  - Предпочтения
  - Производительность интерфейса
  - Поддержка и правила

### Этап 3
- блок `Travel dashboard` внутри `Профиля`,
- локальные `избранные маршруты` с быстрым повтором поиска,
- персональные `напоминания` на основе ближайшей поездки, пассажиров и контактов,
- сводка `история и готовность`:
  - заполненность профиля,
  - основной пассажир,
  - избранные маршруты,
  - частый маршрут по истории бронирований

### Этап 4
- отдельный `command center` внутри `Профиля`,
- `центр уведомлений` с быстрыми действиями:
  - открыть билет,
  - проверить активную заявку,
  - перейти в настройки,
  - открыть пассажиров,
- `журнал поездок` с быстрым входом в активные, завершённые и отменённые заявки,
- отдельная карточка `готовность к рейсу` с процентом готовности и чек-листом.

## Ограничения текущей итерации
- Повтор брони идёт через возврат на `index` с предзаполнением маршрута, а не через отдельный backend-flow.
- Поддержка использует Telegram-ссылку проекта.
- История статусов пока не выносится в отдельный timeline.
- Уведомления и готовность формируются на основе уже доступных данных `dashboard`, без новых backend-событий.

## Критерий успеха
- Пользователь заходит в `Профиль` и сразу видит, есть ли у него ближайшая поездка.
- Основные действия доступны в 1-2 клика.
- Профиль ощущается как личный кабинет поездки, а не просто архив.
- Перед поездкой пользователь видит не только список броней, но и понятный блок `что изменилось / что проверить / куда нажать дальше`.
